# Contributing to BrowserBouncer

First, thank you for your interest in contributing to BrowserBouncer! 

## Code of Conduct
Please be respectful and constructive when engaging with other contributors or opening issues.

## Submitting Pull Requests

1. **Fork the Repository**: Create your own fork of the project and clone it to your local machine.
2. **Create a Branch**: Create a feature branch inside your fork (`git checkout -b feature/your-feature-name`).
3. **Write Code**: Implement your changes. Make sure to adhere to the existing code style. We use SwiftLint to enforce style conventions.
4. **Test Your Changes**: Verify that the app builds without errors and routing works as expected. Check both normal applications and Electron instances.
5. **Document**: If you added a new feature, update the `README.md` and ensure all Swift code has appropriate `///` docstrings.
6. **Submit PR**: Open a Pull Request on the main repository, referencing any issues it resolves and detailing the changes made.

## Opening Issues

We welcome bug reports and feature requests. When opening an issue:
* Verify if the issue already exists.
* Include detailed steps to reproduce the bug.
* Detail your expected behaviour vs what actually happened.
* Include macOS version and any relevant environment details.
